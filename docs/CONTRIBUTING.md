# Contributing to Zoro-DB

Thanks for your interest in contributing to **Zoro-DB**.

We welcome improvements, bug fixes, documentation updates, and new features.

# How to Contribute

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes
4. Submit a Pull Request

Keep changes **small, focused, and easy to review**.

# Development Setup

For instructions on cloning, building, and running Zoro-DB locally, see [DEVELOPMENT.md](./DEVELOPMENT.md)

# Commit Message Guidelines

Use simple prefixes for commits:

| Prefix  | Description                |
| ------- | -------------------------- |
| `feat`  | New feature                |
| `fix`   | Bug fix                    |
| `docs`  | Documentation changes      |
| `tests` | Tests only                 |
| `chore` | Refactor, cleanup, tooling |

Example:

- feat: add vector batch insert
- fix: prevent crash on empty query
- docs: update deployment guide

# Code Style

- Follow the existing code structure
- Keep functions small and readable
- Avoid large unrelated changes in a single PR

# Reporting Issues

If you encounter a bug or want to suggest a feature, please open an issue.

When reporting bugs, include:

- OS version
- Build method (Docker / source)
- Steps to reproduce

# Need Help?

If you get stuck while contributing, open an issue and we'll get back to you as soon as possible.
